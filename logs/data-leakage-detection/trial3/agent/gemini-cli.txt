The audit of `/workspace/pipeline.py` revealed three significant sources of data leakage that resulted in an unrealistically high test accuracy (~97%):

1.  **Global Scaling:** The `StandardScaler` was fitted on the entire dataset before the train/test split, allowing the training set to "see" the distribution (mean and variance) of the test set.
2.  **Global Target Encoding:** The high-cardinality `customer_segment` feature was target-encoded using the mean of the target across the entire dataset. This directly leaked the ground-truth labels of the test set into the features used for training.
3.  **Global Feature Selection:** `mutual_info_classif` was used to select the top 10 features based on the entire dataset, meaning the choice of features was influenced by the test set labels.

I have implemented a corrected pipeline in `/output/pipeline_fixed.py` that strictly separates the training and testing data before any transformation:
- The data is split into train and test sets first.
- **Scaling:** `StandardScaler` is fitted only on the training set and then applied to the test set.
- **Target Encoding:** Category means are calculated using only the training labels. Unseen categories in the test set are filled with the global training target mean.
- **Feature Selection:** Informative features are identified using only the training data.

The corrected pipeline produces a realistic test accuracy of **0.6810**.
